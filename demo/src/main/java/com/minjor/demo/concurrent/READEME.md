# Concurrent(并发)

## 按任务类型

---

### 核心区别：瓶颈在哪里？

*   **IO密集型任务**：瓶颈在于**输入/输出**。任务大部分时间在等待数据从外部设备（如磁盘、网络、数据库、用户输入）读入或写出。CPU大部分时间处于空闲或低负载状态，等待数据就绪。
*   **计算密集型任务**：瓶颈在于**中央处理器**。任务需要持续、高强度的计算，CPU使用率长期接近100%。数据通常已在内存中，几乎没有等待外部设备的延迟。

### 对比表格

| 特性 | IO密集型任务 | 计算密集型任务 |
| :--- | :--- | :--- |
| **主要瓶颈** | **IO速度**（磁盘、网络、数据库） | **CPU计算能力**（主频、核心数、架构） |
| **CPU使用率** | 通常较低，经常在等待 | **持续接近100%** |
| **典型场景** | • 文件读写、复制<br>• 网络请求（API调用、爬虫）<br>• 数据库查询与操作<br>• Web服务器处理请求（大部分时间在等网络和数据库） | • 科学计算（物理模拟、数据分析）<br>• 视频编码/解码、3D图形渲染<br>• 密码学运算（如挖矿）<br
